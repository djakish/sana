The Sana book

A search database, explained from the ground up.

Sana is a small search engine written in Rust. This book builds it up one layer at a time — how data is stored so it survives, how it is shaped so it can be read quickly, and how three different kinds of search are answered over it. Every term is explained the first time it appears. You do not need a database background to start.

What you will be able to do by the end

Follow a single piece of data — a book in a small library — from the moment it is written, through the files it becomes on disk, to the instant a search finds it again. By the last page you will understand what a vector is and why anyone searches with one; what an index is and why it makes search fast; and how a database can live entirely inside something as plain as a folder or a cloud storage bucket.

The running example

Throughout the book we keep a tiny library of five books in a namespace called books. The same five documents reappear at every layer, so you always know what the machinery is actually moving.

How to read it

The chapters are a ladder: each one assumes the one before it and adds a single new idea. Most sections start with the problem in plain language — often the naive thing you might first try, and exactly how it breaks — and only then show how Sana solves it. Diagrams and a handful of interactive panels let you poke at the moving parts. Words in the glossary are defined there if a chapter goes too fast.

The four parts

01 Introduction What a search database is, the three kinds of question people ask of data, and why Sana stores everything in object storage. 02 The durable boundary How writes are made to survive crashes using only an object store's handful of operations: immutable files and compare-and-set. 03 SSTables and LSM trees Why data is kept in sorted, never-edited files, how a lookup finds a key fast, and how background merging keeps reads cheap. 04 The query engine A consistent snapshot, a candidate list that is always rechecked, attribute filters, and full-text search with BM25. 05 Vector search from zero What a vector and an embedding are, why distance means similarity, and how to search millions of them without checking every one. 06 SPFresh and RaBitQ Keeping a vector index fresh as data changes, and making each comparison cheap by testing a compressed sketch before the real one. 07 Running the system Who is allowed to do work, how a process starts and stops safely, where the milliseconds go, and what is honestly still unfinished.
A note on what this is. Sana is an AI-assisted educational project — a clone of the ideas behind turbopuffer, written to be read and learned from, not to run a business on. The code in the repository is the final authority; this book is a guided reading of it.
